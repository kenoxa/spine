# Cross-Provider Hook Formats

Per-provider hook config locations, schema differences, and capability matrix. Prevents silent hook failures when adding new hooks.

## Config Locations

| Provider | Hook config path | Format |
|----------|-----------------|--------|
| Claude Code | `~/.claude/settings.json` (`.hooks` object, written by `generate_claude_hooks()`) | PascalCase events, nested `hooks[]` wrapper |
| Codex | `~/.codex/hooks.json` | PascalCase events, nested `hooks[]` wrapper |
| Cursor | `~/.cursor/hooks.json` | camelCase events, flat entries, `version: 1` required |
| OpenCode | `~/.config/opencode/plugins/spine-hooks.ts` | In-process TS plugin, shell delegation via execFileSync |

**Per-hook gating (Claude+Cursor):** PreToolUse guards (guard-shell, guard-read-large) are always written to `~/.cursor/hooks.json` — double-fire is acceptable for idempotent guards and ensures protection if the Claude plugin fails to load. PostToolUse hooks are omitted everywhere due to the Cursor `additional_context` bug (2.6.x, staff-confirmed — output accepted but not surfaced to model). PreCompact is written only for Cursor-only installs; Claude+Cursor uses the Claude plugin's native prompt hook to avoid duplicate context injection. [E2: `install.sh generate_cursor_hooks():993-1027`]

## Schema Comparison

**Claude Code / Codex** — PascalCase events, nested structure:
```json
{
  "hooks": {
    "PreToolUse": [{"hooks": [{"type": "command", "command": "/path/hook.sh"}]}],
    "PostToolUse": [{"matcher": "Read", "hooks": [{"type": "command", "command": "...", "timeout": 30}]}]
  }
}
```

**Cursor** — camelCase events, flat entries, `version: 1` required:
```json
{
  "version": 1,
  "hooks": {
    "preToolUse": [{"command": "/path/hook.sh", "matcher": "Bash"}],
    "postToolUse": [{"command": "...", "matcher": "Read", "timeout": 30}],
    "preCompact": [{"command": "..."}]
  }
}
```

Differences: no `type` field, no `hooks[]` wrapper, camelCase event names, `version: 1` at root. [E3: `install.sh:983-1061`]

## Capability Matrix

Generated by `hook_supports()` in `install.sh:766-789`. Spine-managed hooks per provider:

| Hook | Claude | Codex | Cursor | OpenCode |
|------|--------|-------|--------|----------|
| `inject-agents-md` (SessionStart) | yes | no | no | no |
| `inject-compact-essentials` (SessionStart) | yes | no | no | no |
| `guard-shell` (PreToolUse) | yes | yes | yes | yes |
| `guard-read-large` (PreToolUse) | yes | yes | yes | yes |
| `inject-types-on-read` (PostToolUse) | yes | no | fires† | yes |
| `check-on-edit` (PostToolUse) | yes | no | fires† | yes |
| `pre-compact` (PreCompact) | yes | no | yes | no |

**`sessionStart` is Claude-only** — Cursor, Codex, and OpenCode load AGENTS.md natively; `inject-agents-md` and `inject-compact-essentials` are suppressed for non-Claude providers. [E2: `install.sh:769-770`]

**Codex PostToolUse gap** — `inject-types-on-read` and `check-on-edit` deferred; Codex PostToolUse fires on Bash only. Tracked in TODO.md.

**† Cursor PostToolUse gap** — hooks execute (E3: `~/.config/spine/logs/hooks.jsonl`) but `additional_context` output is not surfaced to the model. Staff-confirmed bug, Cursor 2.6.x, no fix shipped. See Hook Output Contracts below.

**`preCompact` maps cross-provider** — fires on Claude and Cursor; absent on Codex and OpenCode.

## Hook Output Contracts

Runtime stdout payload formats per hook event. Providers ignore unknown fields.

### PreToolUse

| Field | Provider | Purpose |
|-------|----------|---------|
| `hookSpecificOutput.hookEventName` | Claude Code | Must be `"PreToolUse"` |
| `hookSpecificOutput.permissionDecision` | Claude Code | `"deny"` or `"ask"` — blocks or prompts |
| `hookSpecificOutput.permissionDecisionReason` | Claude Code | User-facing reason string |
| `hookSpecificOutput.additionalContext` | Claude Code | Context injected into model (non-blocking) |
| exit 2 | Cursor, Codex | Blocks execution; stdout/JSON ignored on exit 2 |

Hooks that block emit both `hookSpecificOutput.permissionDecision` (Claude Code) and `exit 2` (Cursor/Codex) simultaneously — each provider reads its own signal.

### PostToolUse

| Field | Provider | Purpose | Status |
|-------|----------|---------|--------|
| `hookSpecificOutput.hookEventName` | Claude Code | Must be `"PostToolUse"` | Working |
| `hookSpecificOutput.additionalContext` | Claude Code | Context injected into model | Working |
| `additional_context` | Cursor (native) | Context injected into model | **Bug: not surfaced (2.6.x, staff-confirmed)** |
| `systemMessage` | — | User-facing warning only; does NOT inject into model context | Do not use for type injection |

**Cursor postToolUse gap:** `inject-types-on-read` and `check-on-edit` fire in Cursor (confirmed E3) but their output is swallowed. No working Cursor alternative for per-read context injection exists. Tracked: forum.cursor.com/t/155689.

**OpenCode postToolUse gap:** OpenCode `tool.execute.after` has no model context injection equivalent. Closed as `not_planned` (#13573).

### SessionStart

| Field | Provider | Purpose |
|-------|----------|---------|
| Plain text stdout | Claude Code | Injected as session context |
| `additional_context` | Cursor (native) | Static session context (confirmed working) |
| `{}` (empty JSON) | Cursor (command hook bail) | Silent no-op — all fields optional per Cursor docs |

**Claude-only sessionStart hooks** (`inject-agents-md`, `inject-compact-essentials`) must bail with `printf '{}'` when `SPINE_PROVIDER_IS_CURSOR=1`. Detection: `_env.sh` exports `SPINE_PROVIDER_IS_CURSOR=1` when `CURSOR_PLUGIN_ROOT` is set (confirmed E3: Cursor 3.0.9). `CURSOR_AGENT` is absent in hook subprocesses — it applies to CLI/terminal context only.

## _env.sh Implementation Constraints

### Cursor env var detection — CLI vs hook subprocess

`CURSOR_AGENT=1` is set in Cursor CLI/terminal sessions only — absent in hook
subprocesses spawned via the Cursor Claude plugin. `CLAUDECODE` is also absent
in hook subprocesses; negative detection is unreliable.

Vars confirmed present in Cursor 3.0.9 hook subprocess env (E3: `~/.config/spine/logs/hooks.jsonl`):
`CURSOR_PLUGIN_ROOT`, `CURSOR_VERSION`, `CURSOR_USER_EMAIL`, `CURSOR_WORKSPACE_LABEL`,
`CURSOR_LAYOUT`.

Canonical detection pattern (implemented at `hooks/_env.sh:52`):
```sh
{ [ -n "${CURSOR_PLUGIN_ROOT:-}" ] || [ -n "${CURSOR_VERSION:-}" ]; } \
  && export SPINE_PROVIDER_IS_CURSOR=1 || true
```
`CURSOR_PLUGIN_ROOT` is most precise (plugin execution path); `CURSOR_VERSION` is
broader fallback.

### wc -c redirection — shell error not suppressed by 2>/dev/null

`wc -c < file 2>/dev/null` does not silence stderr when `file` is absent. The shell
performs the `< file` redirect before exec-ing `wc`; if the file is missing, the
shell emits the error on stderr — `2>/dev/null` covers only the child process, not
the shell's own redirect failure.

Safe pattern: subshell wrap.
```sh
bytes=$( (wc -c < "$file") 2>/dev/null || echo 0)
```
The subshell makes the redirect a child-process operation; `2>/dev/null` now covers
both the redirect error and `wc`'s stderr. [E3: verifier confirmed stderr leak on macOS]

### exec boundary — functions lost, exported vars survive

`_env.sh` calls `exec /bin/sh "$_spine_cmd"` at line 80, replacing the process.
Shell functions defined before exec are lost. Exported env vars survive (POSIX
inheritance). `export -f` (function export) is a bashism — unavailable in `#!/bin/sh`.

Any cross-hook state from `_env.sh` must use `export VAR=value`, not functions.
Check in hook body with `[ "${FLAG:-}" = "1" ]`. Existing vars that follow this
pattern: `SPINE_ENV_LOADED`, `PATH`, `SPINE_HOOKS_DIR`, `SPINE_PROVIDER_IS_CURSOR`.
Vars sourced from `~/.config/spine/.env` must also use `export`; bare assignments
survive in `_env.sh`'s local env but are dropped by the `exec` handoff. [E3]

## Cursor-Specific Events

Cursor supports additional event types not available in Claude Code or Codex:
`subagentStart`, `subagentStop`, `postToolUseFailure`, `beforeShellExecution`, `beforeReadFile`, `afterFileEdit`.

Spine does not currently use these — available for future hooks.

## Adding a New Hook

1. Add to `hook_supports()` for each supported provider/event pair.
2. Add generation logic to `generate_claude_hooks()`, `generate_codex_hooks()`, `generate_cursor_hooks()` using the correct schema for each provider.
3. Re-run `install.sh` — generators strip and rewrite provider configs with updated entries.
